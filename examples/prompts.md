# Prompt Examples

These prompts can be copied directly into Codex.

## Planning only

```text
Use $illustrations-to-explain-things, but do not generate images yet.
Analyze the article below and propose a shot list of about 5 illustrations.
For each one, include:
- where it should appear
- the theme
- the core idea
- the structure type
- what Xiaohei is doing
- suggested elements
- suggested handwritten labels in English

<paste article>
```

## Article illustration set

```text
Use $illustrations-to-explain-things to generate 4 article illustrations for the piece below.
Requirements: 16:9 landscape, pure white background, black hand-drawn line art, sparse red/orange/blue handwritten labels in English.
Each image should explain only one core structure. Do not make PPT infographics or cute mascot art.

<paste article>
```

## Long-form illustration strategy

```text
Use $illustrations-to-explain-things to plan illustrations for this long article.
Do not spread images evenly. Only pick cognitive anchors such as core judgments, input/output loops, before/after shifts, common traps, and handoff paths.
Default to 6-8 images. Output the shot list first and do not generate images yet.

<paste article>
```

## One concept, one image

```text
Use $illustrations-to-explain-things to create one 16:9 article illustration for this idea:

Trust is not claimed out loud; it is laid down piece by piece through evidence.

Keep it strange but clean. Xiaohei must perform the core action.
Use no more than 5 short handwritten labels in English.
```

## Workflow theme

```text
Use $illustrations-to-explain-things to create one illustration for:
"Turn one raw source asset into three kinds of content: reach, trust, and conversion."
Do not draw a formal flowchart and do not copy the old one-input-many-outputs example.
Invent a fresh low-tech metaphor and make Xiaohei perform the central action.
```

## Edit: remove a title

```text
Use $illustrations-to-explain-things to edit this image.
Remove the top-left title "Workflow / Flowchart" and its underline.
Keep everything else unchanged.
Do not add any new text or objects.
```

## Edit: make Xiaohei matter more

```text
Use $illustrations-to-explain-things.
This image is headed in the right direction, but Xiaohei still feels decorative.
Keep the core idea the same and regenerate it so Xiaohei becomes the one actually driving the structure.
Make it a bit stranger while keeping it pure white, clean, and sparse.
```

## Generate a mini style set

```text
Use $illustrations-to-explain-things to generate 5 Xiaohei article-illustration samples.
Cover these themes: information overload, product validation, content compounding, solo business systems, and trust building.
Generate each image separately rather than composing them into one sheet.
```
